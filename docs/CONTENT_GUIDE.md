# Content and Continuity Guide

## Purpose

This guide keeps many Markdown documents usable as one world model. Each page should be readable by a contributor and retrievable by an AI runtime without hidden dependencies.

## One owner per fact

Store a fact in its most specific authoritative file and link to it elsewhere. A character page owns motives and limits; a location page owns local access and atmosphere; a faction page owns membership and strategy; gameplay pages own reusable rules; campaign saves own playthrough events.

## Canon and invention

Write canon summaries conservatively. Where the series leaves a gap, use “campaign default,” “commonly believed,” or “not established.” Original developments should follow from wartime institutions, geography, resources, and motives rather than later continuity.

## Useful detail

Prioritize information that changes a scene: who wants what, what blocks them, who knows, what evidence exists, how long it takes, what it costs, and what happens without intervention. Decorative detail should reinforce culture, mood, or choice.

## Character integrity

Characters are not player rewards. Define obligations, information sources, loyalties, boundaries, coping patterns, and capacity to change. Separate narrator truth from what the character can say.

## Cross-file review

When changing a major fact, review related character, location, faction, nation, timeline, and world-state pages. Update indexes when adding content. Avoid hard-coding a campaign outcome into a reusable reference page.

## Files and links

Use descriptive filenames and preserve each folder's current naming convention. Use relative links with exact case. Start with one H1 and use H2 headings for major retrieval sections.
