# Runtime and Retrieval Guide

## Context assembly

Begin with `SYSTEM_PROMPT.md`, the canon boundary, current campaign state, and active scene. Retrieve the current location, relevant nation and faction, participating character profiles, and only the gameplay rules needed for the intended action. Irrelevant profiles increase continuity errors.

## State ownership

Repository documents provide rules and defaults. A campaign save owns current truth after play begins. If the save says a bridge collapsed, a leader resigned, or a relationship ended, that event overrides the reusable default.

## Turn pipeline

1. Parse intent without inventing unspoken consent or dialogue.
2. Retrieve rules, scene facts, character constraints, and recent memory.
3. Test feasibility, knowledge access, opposition, and elapsed time.
4. Resolve and narrate observable consequences.
5. Apply state changes with reasons and timestamps.
6. Advance relevant clocks and scheduled events.
7. Validate location, knowledge, inventory, injury, and relationships.

## Memory tiers

- **Scene:** positions, hazards, participants, and declared intentions.
- **Recent:** promises, discoveries, injuries, and likely near-term context.
- **Entity:** durable beliefs and relationship history.
- **World:** dated public events and permanent consequences.

Summarize old detail instead of carrying transcripts. Preserve exact commitments, boundaries, debts, and unresolved clues.

## Validation questions

Is everyone in a plausible place? Does each speaker know this? Did enough time pass? Were costs applied? Did an NPC abandon a core value without an arc? Did the response decide the player's feelings? Did a rumor become fact without evidence?
